The point

A general model will happily invent an API that doesn’t exist. This one can’t: every answer is built only from retrieved documentation — or it tells you it’s beyond the docs and points you to the source.

What makes it different

Grounded, current, and honest about its limits.

Citations

Grounded, or it refers you out

Every claim traces to a retrieved doc section; citations open the exact page and anchor. Source-code internals get a DeepWiki / GitHub referral — never a guess dressed as fact.

Freshness

The index heals itself

After answering, it revalidates the cited pages against the live docs. If they drifted, it re-embeds the changed sections in place and corrects the answer — no rebuild, no stale cache.

Transparency

You watch it think

The reasoning streams live in grey — what it searched, what it found — then resolves into the answer in black. Honest progress for a payload that can only arrive whole.

Retrieval

Hybrid search, lean stack

Dense pgvector + keyword tsvector, RRF-merged over Neon Postgres. Embeddings run on a local 384-dim model on CPU: no embedding API, no quota, no cost.

Resilience

Won’t fall over

A provider fallback chain — OpenRouter → Anthropic → Gemini — with per-model cooldowns. One rate-limited or broken key self-heals to the next instead of taking the app down.

Discipline

Checked before it ships

Every snippet is statically checked — it must parse, import only torch/stdlib, and mention only symbols that exist in the index. A failed check triggers one regeneration, not a bluff.

The pipeline

From the docs site to a cited answer, in four stages.

Data flows one way. Each stage is a package in the repo, documented on its own.

01

ingest

Discover every page (Sphinx objects.inv + sitemaps), crawl it, and heading-chunk the docs site into an on-disk snapshot.

02

index

Embed into Neon (pgvector + tsvector). Hybrid retrieval returns pointers; content is hydrated straight from the database.

03

agent

Guard → route → a single grounded pass or a tool loop (search_docs / read_page / ask_source) → a validated answer.

04

app

A Gradio Space streams the reasoning trace and renders the answer with clickable citations — then quietly checks the docs are still current.

Self-host

Run or deploy your own

Embeddings run locally on CPU, so building the index needs only a Neon connection string.

# 1 · install
git clone https://github.com/eliezeravihail/TorchDocsAgent
cd TorchDocsAgent && pip install -e .

# 2 · build the index (crawl + embed, CPU, minutes)
export NEON_URL=postgres://...
python scripts/build_index.py

# 3 · run locally — or deploy to HF Spaces
python -m app.main
